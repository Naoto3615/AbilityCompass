# Ability Compass 要件書

**プロジェクト名：** Ability Compass  
**バージョン：** 1.0（Phase 1 + Phase 2 完了）  
**最終更新：** 2026年5月27日  
**開発言語：** Python 3.12 / Django 4.2

---

## 1. プロジェクト概要

### 1.1 目的

発達障害（ADHD・ASD・LD など）を持つ子供たちの「強み」にフォーカスし、
将来の可能性を広げることを目的としたウェブアプリケーション。
欠点の矯正ではなく、個性を才能として伸ばすことをコアコンセプトとする。

### 1.2 ターゲットユーザー

| ユーザー | 説明 |
|---------|------|
| 子供（3歳〜成人） | 発達障害を持つ本人（保護者と一緒に利用するケースも想定） |
| 保護者・支援者 | 子供の成長をサポートする親・教師・支援員 |

### 1.3 開発フェーズ

| フェーズ | 内容 | 状態 |
|---------|------|------|
| Phase 1 | 適性診断 + キャリアロードマップ（MVP） | ✅ 完了 |
| Phase 2 | ユーザー認証 + デイリータスク + 感情ログ + 保護者ダッシュボード | ✅ 完了 |
| Phase 3 | AI 会話練習 + ポートフォリオ生成 | 🔜 未着手 |
| Phase 4 | 独自 AI モデルのファインチューニング | 🔜 未着手 |
| Phase 5 | モバイルアプリ化（React Native） | 🔜 未着手 |

## 6. 機能仕様

### 6.1 適性診断（Phase 1）

**概要**  
15の質問に1〜5のスコアで回答し、AIが強みと向いている職業を分析する。

**診断質問（15問・強みフォーカス）**

| No | カテゴリ | 質問内容 |
|----|---------|---------|
| 1 | focus（集中力） | 好きなことや気になることに対して、時間を忘れて集中することがある |
| 2 | pattern（パターン認識） | 数字・図形・パターンを見ると面白いと感じる |
| 3 | creative（想像力） | 頭の中でイメージや絵を思い浮かべるのが得意だ |
| 4 | movement（身体表現） | 体を動かすことや手を使って何かを作ることが好きだ |
| 5 | music（音楽） | 音楽を聴いたり、リズムや音の変化に気づくことが多い |
| 6 | nature（自然） | 動物や植物・自然現象に強い興味を持っている |
| 7 | empathy（共感力） | 他の人や動物の気持ちを察したり、共感することが多い |
| 8 | rule（規律） | ルールや手順を守ることに強いこだわりを感じる |
| 9 | detail（注意力） | 細かいところや他の人が見落とすようなことに気づく |
| 10 | story（物語） | 物語を読んだり話を作ったりするのが好きだ |
| 11 | logic（論理） | 「なぜ？」「どうして？」と理由を考えるのが好きだ |
| 12 | social（独立性） | みんなで協力して何かを達成するよりも、一人で取り組む方が好きだ |
| 13 | memory（記憶） | 好きなテーマについての知識や情報をたくさん覚えている |
| 14 | system（体系化） | 物事を整理したり、順番を決めたりするのが好きだ |
| 15 | visual（視覚思考） | 説明を読むより、図や絵で見た方が理解しやすい |

**診断結果**
- 強み TOP3（タイトル・説明・絵文字）
- 向いている職業 TOP3（マッチ度スコア・理由）
- ポジティブなメッセージ

**AI 処理**
- OpenAI GPT-4o-mini に回答を渡し JSON で結果を受け取る
- API キーなし・エラー時はルールベースのフォールバック分析

### 6.2 キャリアロードマップ（Phase 1）

**概要**  
14種類の職業から選択し、6段階の年齢フェーズ別に「今何をすべきか」を表示する。

**対応職業（14種類）**

プログラマー・エンジニア / デザイナー・アーティスト / 音楽家・作曲家 / 科学者・研究者 / 作家・ライター / 数学者・データサイエンティスト / ゲームクリエイター / シェフ・料理研究家 / 動物の専門家 / 建築家・空間デザイナー / 教師・支援員 / スポーツ選手・コーチ / 写真家・映像作家 / 起業家・経営者 / 医師・医療専門家

**年齢フェーズ（6段階）**

| フェーズ | 対象年齢 |
|---------|---------|
| 幼少期 | 3〜6歳 |
| 小学校低学年 | 7〜9歳 |
| 小学校高学年 | 10〜12歳 |
| 中学生 | 13〜15歳 |
| 高校生 | 16〜18歳 |
| 大学・社会人 | 19歳〜 |

**ロードマップ内容（カテゴリ別）**
- 学習・スキル
- 人間性・心の成長
- 日々の習慣
- コミュニケーション
- 強みを活かす活動

加えて「このフェーズの最重要目標」と「励ましのメッセージ」を表示。

**キャッシュ機能**  
職業 × 年齢の組み合わせごとに DB にキャッシュ。
同一組み合わせは API を呼ばずキャッシュを返す。

### 6.3 ユーザー認証（Phase 2）

**子供アカウント登録項目**
- ニックネーム
- アバター絵文字（12種類から選択）
- 年齢フェーズ
- 目標職業（任意、あとで変更可）
- ログインID・パスワード

**保護者アカウント登録項目**
- ログインID・パスワード
- お子様のログインID（任意・後からでも紐付け可）

**ログイン後のリダイレクト**
- 子供アカウント → `/daily/`（ダッシュボード）
- 保護者アカウント → `/accounts/parent/dashboard/`

### 6.4 デイリータスク・ダッシュボード（Phase 2）

**概要**  
子供が毎日ログインすると、その日のミッション（タスク）が5件表示される。
ゲーミフィケーションにより継続モチベーションを維持する。

**タスク生成**
- 1日1回、OpenAI API でその日のタスクを生成
- 職業目標と年齢フェーズに基づいたパーソナライズ
- API なし・エラー時はフォールバックのプリセットタスクを表示
- タスクは DB に保存（翌日は再生成）

**ゲーミフィケーション**

| 要素 | 仕様 |
|------|------|
| ポイント | タスク完了で 10〜15 pt 獲得 |
| レベル | 5段階（累計ポイントに応じて昇格） |
| バッジ | 6種類（条件達成時にモーダル通知） |

**バッジ一覧**

| バッジ名 | 条件 |
|---------|------|
| 👣 初めての一歩 | 初回タスク完了 |
| 🔥 7日連続 | 7日連続でタスク完了 |
| 📔 気持ちの記録家 | 感情ログを5日間記録 |
| ⭐ 100ポイント達成 | 合計100pt獲得 |
| 🏆 500ポイント達成 | 合計500pt獲得 |
| 🌈 バランスの達人 | 全5カテゴリのタスクを完了 |

**ダッシュボード表示要素**
- 今日の進捗バー（完了数/全体数）
- タスクカード（カテゴリ・内容・ポイント・チェックボタン）
- タスク完了はAjax処理（ページ遷移なし）
- 1週間の棒グラフ
- 今日の気持ち（感情ログへのリンク）
- 獲得バッジ一覧

### 6.5 感情ログ（Phase 2）

**概要**  
毎日の気持ちを絵文字で記録し、AIがトレンドを可視化する。
感情の言語化が苦手な子供でも、絵文字・色で直感的に選べるUI設計。

**記録項目**
- 気持ちスコア（1〜5、絵文字選択式）
- 今日あったこと・気づいたこと（任意テキスト）

**表示機能**
- 過去2週間の気持ちトレンドグラフ（Chart.js 折れ線グラフ）
- 最近の記録リスト（絵文字・日付・ラベル）

### 6.6 保護者ダッシュボード（Phase 2）

**概要**  
保護者がお子様の成長を確認し、適切なサポートを行えるダッシュボード。

**表示内容**
- 子供プロフィール（ニックネーム・年齢・目標職業・レベル）
- 今週のタスク達成率（ドーナツグラフ）
- 直近7日間の気持ちトレンドグラフ
- AIによる保護者向けサポートアドバイス
- 獲得バッジ一覧
- 最近のタスク一覧（完了・未完了）

**AIアドバイス**  
子供のタスク完了率・感情スコアを渡し、GPT-4o-mini が保護者向けの
具体的なサポートアドバイスを生成する（200文字以内）。

## 8. AI 連携仕様

### 8.1 使用モデル

| 機能 | モデル | 呼び出しタイミング |
|------|--------|-----------------|
| 適性診断分析 | GPT-4o-mini | 診断フォーム送信時（1回） |
| ロードマップ生成 | GPT-4o-mini | 職業×年齢の初回表示時（キャッシュあり） |
| デイリータスク生成 | GPT-4o-mini | 当日初回ダッシュボード表示時（1日1回） |
| 保護者アドバイス | GPT-4o-mini | 保護者ダッシュボード表示時 |

### 8.2 フォールバック設計

OpenAI API キー未設定・エラー時でもアプリが動作するよう、
すべての AI 機能にルールベースのフォールバックを実装済み。

| 機能 | フォールバック内容 |
|------|-----------------|
| 診断分析 | スコア上位カテゴリから強み・職業を自動マッピング |
| ロードマップ | 年齢フェーズ別プリセットデータを返却 |
| デイリータスク | 年齢フェーズ別プリセットタスク5件を返却 |
| 保護者アドバイス | 固定の汎用アドバイステキストを返却 |

### 8.3 プロンプト設計方針

- 発達障害への理解を前提とした専門的なシステム指示
- 欠点・困難ではなく「強み・可能性」にフォーカスした表現を指示
- JSON レスポンス形式を強制（`response_format: json_object`）
- 文字数上限を明示して出力ブレを防止

## 11. 起動方法

```bash
# 依存パッケージのインストール
pip install -r requirements.txt

# 環境変数の設定
cp .env.example .env
# .env の OPENAI_API_KEY に API キーを入力（任意）

# データベースの初期化
python manage.py migrate

# 開発サーバーの起動
python manage.py runserver 8000
```

ブラウザで `http://127.0.0.1:8000/` にアクセス。
